VoIP.ms Webhook Monitor - Test Page

Setup Instructions:
  1. Upload webhook.php to your cPanel
  2. Update SLACK_BOT_TOKEN and SLACK_CHANNEL in the PHP file
  3. Enter your webhook URL below
  4. Select a test scenario and click "Send Test Webhook"
  5. Check your Slack channel for the notification

Send Test Webhook

Expected Behavior:
  • ALERTS - New York incidents, components, maintenance
  • ALERTS - Service-wide issues (DDoS, billing, SIP, DNS, auth)
  • ALERTS - Critical severity incidents (any location)
  • IGNORES - Other locations (London, LA, etc.)

Test Scenarios Reference

Scenario Type Expected Result Slack Header
NY Incident - Major Incident ALERT New York Server Incident
NY Component Update Component ALERT New York Server Component Update
NY Maintenance Maintenance ALERT Scheduled Maintenance Alert
DDoS Attack Incident ALERT Service-Wide Alert
Billing Issue Incident ALERT Service-Wide Alert
Critical Severity Incident ALERT Service-Wide Alert
London Incident Incident IGNORE N/A (no alert)
Debugging Tips:
  • Check webhook.log in your cPanel File Manager for detailed logs
  • Log entries show why each webhook was processed or ignored
  • Verify your Slack bot is invited to the channel: /invite @YourBotName
